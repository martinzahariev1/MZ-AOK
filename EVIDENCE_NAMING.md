# Evidence Naming

Use consistent filenames so documents can be sorted, searched, and cited.

## Standard Format

`YYYY-MM-DD_Source_Type_Short-description_Language_Status.ext`

## Examples

- `2026-06-23_AOK_Letter_Contribution-demand_DE_original.pdf`
- `2026-06-24_Employee-Name_WhatsApp_Payment-request_BG_copy.png`
- `2026-06-25_Bank_Statement_June-transactions_DE_original.pdf`
- `2026-06-26_Court_Filing_Objection_DE_final.pdf`

## Rules

- Use ISO dates: `YYYY-MM-DD`.
- Use short source names without private extra detail.
- Use hyphens inside descriptions, not spaces.
- Use one language code: `BG`, `DE`, `EN`, or `MIX`.
- Use one status: `original`, `copy`, `translated`, `annotated`, `draft`, or `final`.
- Preserve the original extension.
