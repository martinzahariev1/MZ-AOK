# Case Guide

This repository stores case evidence, documents, entity records, financial traces, legal materials, analysis, reports, automation notes, and archived material.

## Working Principles

- Keep original evidence unchanged whenever possible.
- Store files in the most specific folder available.
- Record every important item in the relevant dashboard CSV.
- Use clear dates, source names, document types, and short descriptions in filenames.
- Keep personal notes separate from source documents when possible.

## Required Filename Pattern

Use this pattern for evidence and documents:

`YYYY-MM-DD_Source_Type_Short-description_Language_Status.ext`

Example:

`2026-06-23_AOK_Email_Insurance-response_DE_original.pdf`

## Languages

- `BG` for Bulgarian
- `DE` for German
- `EN` for English
- `MIX` for mixed-language material

## Status Values

- `original`
- `copy`
- `translated`
- `annotated`
- `draft`
- `final`
